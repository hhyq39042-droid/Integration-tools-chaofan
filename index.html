部署指南 & 更新说明

本工具中心部署在 GitHub Pages（免费），团队成员只需浏览器 + 访问链接，无需任何账号。 管理员在 GitHub 更新工具文件后，成员点击「刷新最新版」即可同步。
首次部署（仅做一次）
1

注册 GitHub 账号 免费

访问 github.com 注册，用于托管工具文件。团队成员不需要账号。

2

创建新仓库

登录后点击右上角 + → New repository
仓库名随意（如 my-tools），勾选 Public（公开），点击创建。

3

上传全部文件 5个文件

进入仓库页面 → 点击 uploading an existing file 或 Add file → Upload files
将以下 5 个文件一起拖入上传区域：

index.html 审核系统完整版.html 售价核查工具.html 新品成功率计算工具.html 亚马逊周报生成器_v7.html

上传后点击 Commit changes 保存。

4

开启 GitHub Pages

进入仓库 → 顶部 Settings → 左侧 Pages
在 Source 下拉选 Deploy from a branch，Branch 选 main，点击 Save。
等待约 1 分钟，页面会出现你的访问链接：

https://你的用户名.github.io/仓库名/
5

分享链接给团队

将上一步得到的链接发给所有团队成员，直接用浏览器打开即可，不需要注册任何账号。

更新工具（日常操作）
1

登录 GitHub → 进入仓库页面

2

点击要更新的工具文件名，右上角点击铅笔图标 Edit 或直接 Upload files 重新上传同名新文件

3

点击 Commit changes 保存

4

等待约 30 秒 ~ 1 分钟，GitHub Pages 自动更新

5

通知团队成员点击工具中心右上角 「刷新最新版」 按钮，即可加载新版本

添加新工具
1

将新工具 HTML 文件上传到 GitHub 仓库

2

用文本编辑器打开 index.html，在顶部 TOOLS 数组中添加一行：
{ id: 5, name: '新工具名称', file: '新工具文件名.html' },

3

重新上传修改后的 index.html 到 GitHub，提交后即生效

团队成员无需账号，只需浏览器打开链接即可使用
