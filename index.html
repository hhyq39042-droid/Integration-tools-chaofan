说明 & 更新指南

关于工具内置数据
⚠ 本地文件打开的数据（file://）与网页版数据（GitHub Pages）互相独立，无法迁移。
首次通过工具中心使用时，需要在工具内重新上传或录入一次数据，此后数据会在同一浏览器下自动保留。
✓

同一浏览器、同一链接打开 → 数据自动保留，无需重复操作

✓

换浏览器或换电脑 → 需重新录入一次（数据存在各自浏览器中）

✓

清除浏览器缓存/数据 → 数据会丢失，请谨慎操作

自动更新通知
1

系统每隔 3分钟 自动检查所有工具及工具中心是否有更新

2

工具内容更新 → 标签红点 + 黄色横幅 → 点「立即更新」

3

新增工具 → 蓝色横幅 → 点「刷新页面」获取新标签

更新已有工具（GitHub操作）
1

GitHub 仓库 → 点击工具文件 → 右上角 上传图标 ↑ → 上传新版（文件名须相同）

2

点击 Commit changes，等约30秒~1分钟，系统自动提示成员更新
